# AI 코드 거버넌스 워크플로우

> 팀에서 AI 생성 코드의 품질, 보안, 라이선스를 체계적으로 관리하는 정책 수립과 운영 가이드

## 개요

AI 코딩 에이전트를 팀에 도입하면 생산성은 올라가지만 새로운 위험도 따라와요. AI가 생성한 코드에 보안 취약점이 숨어 있을 수 있고, 학습 데이터에서 온 코드가 라이선스 문제를 일으킬 수도 있어요. "일단 쓰고 나중에 정리하자"는 접근은 기술 부채와 법적 리스크를 동시에 키우게 됩니다.

이 워크플로우는 AI 코드 거버넌스를 **정책 수립 → 리뷰 기준 → 자동화 → 감사 추적** 4단계로 나눠서 팀이 바로 적용할 수 있는 실전 프레임워크를 제공해요.

## 사전 준비

- Git 기반 코드 관리 (GitHub/GitLab)
- CI/CD 파이프라인 (GitHub Actions, GitLab CI 등)
- AI 코딩 도구 1개 이상 (Claude Code, Cursor, Copilot 등)
- 팀 규모 3명 이상

## 1단계: AI 코드 정책 수립

### 정책 문서 템플릿

프로젝트 루트에 `AI_CODE_POLICY.md`를 만들어요:

```markdown
# AI Code Policy

## 허용 범위
- AI 코드 생성: 허용 (리뷰 필수)
- AI 코드 자동 커밋: 금지 (사람 검수 후 커밋)
- AI 테스트 생성: 허용 (커버리지 80% 이상 유지)

## 금지 사항
- AI 생성 코드를 리뷰 없이 프로덕션 배포
- 민감 데이터(API 키, 개인정보)를 AI 프롬프트에 입력
- AI 생성 코드의 라이선스 출처 미확인 상태로 머지

## 리뷰 기준
- 보안: OWASP Top 10 체크리스트 통과
- 품질: 팀 린트 규칙 + 테스트 커버리지 기준 충족
- 라이선스: GPL/AGPL 코드 혼입 여부 확인
```

### 위험 등급 분류

| 등급 | 대상 코드 | 리뷰 수준 | 예시 |
|------|----------|----------|------|
| **높음** | 인증, 결제, 개인정보 처리 | 시니어 2명 리뷰 + 보안 스캔 | OAuth 플로우, 결제 API |
| **중간** | 비즈니스 로직, API 엔드포인트 | 1명 리뷰 + 자동 테스트 | CRUD API, 데이터 변환 |
| **낮음** | 설정, 문서, 유틸리티 | 1명 리뷰 | README, 헬퍼 함수 |

## 2단계: AI 코드 리뷰 체크리스트

### PR 템플릿에 AI 관련 항목 추가

`.github/PULL_REQUEST_TEMPLATE.md`에 다음을 포함해요:

```markdown
## AI 사용 여부
- [ ] 이 PR에 AI 생성 코드가 포함되어 있습니다
- [ ] AI 도구: (사용한 도구명)
- [ ] AI 생성 코드를 직접 검토했습니다

## AI 코드 체크리스트 (AI 사용 시)
- [ ] 하드코딩된 시크릿/토큰 없음
- [ ] SQL 인젝션/XSS 취약점 없음
- [ ] 불필요한 외부 패키지 추가 없음
- [ ] 라이선스 호환성 확인
- [ ] 테스트 코드 포함
```

### 리뷰어 가이드

AI 생성 코드를 리뷰할 때 특별히 확인할 포인트:

```yaml
# .github/ai-review-guide.yml
focus_areas:
  security:
    - "입력값 검증이 빠져있지 않은가?"
    - "에러 메시지에 내부 정보가 노출되지 않는가?"
    - "인증/인가 체크가 올바른가?"
  quality:
    - "엣지 케이스 처리가 되어 있는가?"
    - "에러 핸들링이 적절한가?"
    - "불필요하게 복잡한 구현은 없는가?"
  license:
    - "특정 오픈소스 프로젝트와 유사한 코드가 있는가?"
    - "새로 추가된 패키지의 라이선스는 호환되는가?"
```

## 3단계: CI/CD 자동화 파이프라인

### GitHub Actions 거버넌스 워크플로우

```yaml
# .github/workflows/ai-code-governance.yml
name: AI Code Governance Check

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  security-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      # 시크릿 스캐닝
      - name: Secret Detection
        uses: trufflesecurity/trufflehog@main
        with:
          extra_args: --only-verified
      
      # 의존성 취약점 검사
      - name: Dependency Audit
        run: |
          npm audit --audit-level=high || true
          pip-audit --strict || true
      
      # SAST (정적 분석)
      - name: Static Analysis
        uses: github/codeql-action/analyze@v3

  license-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      # 라이선스 호환성 검사
      - name: License Compliance
        run: |
          npx license-checker --failOn "GPL-2.0;GPL-3.0;AGPL-3.0"

  code-quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      # 린트 + 포맷 검사
      - name: Lint Check
        run: |
          npm run lint
          npm run format:check
      
      # 테스트 커버리지
      - name: Coverage Gate
        run: |
          npm test -- --coverage
          # 커버리지 80% 미만이면 실패
```

### AI 코드 탐지 라벨링

PR에 AI 코드가 포함되었는지 자동 탐지하고 라벨을 붙이는 방법:

```yaml
# .github/workflows/ai-label.yml
name: AI Code Label

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  detect-ai-patterns:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      
      - name: Check for AI tool markers
        id: ai-check
        run: |
          # PR diff에서 AI 도구 흔적 탐지
          DIFF=$(git diff origin/main...HEAD)
          
          # 일반적인 AI 생성 패턴 체크
          if echo "$DIFF" | grep -qiE "copilot|generated by|auto-generated"; then
            echo "ai_detected=true" >> "$GITHUB_OUTPUT"
          fi
          
          # 대량 코드 추가 체크 (한 파일에 200줄+)
          LARGE_ADDITIONS=$(git diff --numstat origin/main...HEAD | \
            awk '$1 > 200 {print $3}')
          if [ -n "$LARGE_ADDITIONS" ]; then
            echo "large_addition=true" >> "$GITHUB_OUTPUT"
          fi
      
      - name: Add label
        if: steps.ai-check.outputs.ai_detected == 'true'
        uses: actions/github-script@v7
        with:
          script: |
            github.rest.issues.addLabels({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: context.issue.number,
              labels: ['ai-generated']
            })
```

## 4단계: 감사 추적과 메트릭

### 감사 로그 구조

AI 코드 관련 이벤트를 추적하는 로그 형식:

```jsonl
{"timestamp":"2026-03-26T14:00:00Z","event":"ai_code_merged","pr":123,"author":"dev1","tool":"claude-code","files_changed":3,"lines_added":150,"reviewer":"dev2","security_scan":"pass","license_check":"pass"}
{"timestamp":"2026-03-26T15:00:00Z","event":"ai_code_rejected","pr":124,"author":"dev1","tool":"cursor","reason":"security_vulnerability","details":"SQL injection in query builder"}
```

### 주간 거버넌스 대시보드

```bash
#!/bin/bash
# scripts/governance-report.sh

echo "=== AI 코드 거버넌스 주간 리포트 ==="
echo ""

# 이번 주 AI 관련 PR 통계
TOTAL=$(gh pr list --state merged --search "label:ai-generated" \
  --json number --jq 'length')
REJECTED=$(gh pr list --state closed --search "label:ai-generated -is:merged" \
  --json number --jq 'length')

echo "머지된 AI PR: $TOTAL건"
echo "거절된 AI PR: $REJECTED건"
echo "수용률: $(( TOTAL * 100 / (TOTAL + REJECTED) ))%"
echo ""

# 보안 스캔 결과 요약
echo "보안 이슈 발견: $(grep -c 'security_vulnerability' governance.log)건"
echo "라이선스 문제: $(grep -c 'license_violation' governance.log)건"
```

### 핵심 메트릭 (KPI)

| 메트릭 | 목표 | 측정 방법 |
|--------|------|----------|
| AI 코드 수용률 | 85%+ | 머지된 AI PR / 전체 AI PR |
| 보안 이슈 발견률 | 회귀 0건 | 프로덕션 보안 인시던트 추적 |
| 리뷰 처리 시간 | 24시간 내 | PR 생성~머지 시간 |
| 라이선스 위반 | 0건 | CI 라이선스 체크 결과 |
| 테스트 커버리지 | 80%+ | CI 커버리지 리포트 |

## 커스터마이징

| 설정 | 기본값 | 조정 가이드 |
|------|--------|------------|
| 위험 등급 기준 | 3단계 | 팀 규모와 도메인에 맞게 세분화 |
| 리뷰어 수 | 높음=2, 중간=1 | 팀 크기에 비례하여 조정 |
| 커버리지 기준 | 80% | 레거시 프로젝트는 60%부터 시작 |
| 라이선스 블랙리스트 | GPL/AGPL | 회사 법무팀과 협의하여 결정 |
| 감사 로그 보관 | 90일 | 규제 요건에 따라 연장 |

## 문제 해결

| 문제 | 해결 |
|------|------|
| "AI 라벨이 자동으로 안 붙어요" | GitHub Actions 워크플로우 권한에 `issues: write` 추가 |
| "라이선스 체크가 너무 엄격해요" | `failOn` 목록에서 허용할 라이선스를 제외하거나 allowlist 사용 |
| "보안 스캔이 오탐이 많아요" | `.trufflehog.yml`에 예외 패턴 등록, `--only-verified` 옵션 사용 |
| "리뷰 병목이 생겨요" | CODEOWNERS에 AI 코드 전문 리뷰어 지정, 위험 등급별 리뷰어 분산 |
| "감사 로그가 안 쌓여요" | CI 워크플로우에서 아티팩트 업로드 단계 확인 |
