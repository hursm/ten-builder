# 플레이북 17: AI 프로토타이핑

> 아이디어를 30분 안에 동작하는 프로토타입으로 만드는 6단계 워크플로우

## 언제 쓰나요?

- 새로운 사이드 프로젝트 아이디어를 빠르게 검증하고 싶을 때
- 팀 미팅 전에 동작하는 데모를 만들어야 할 때
- 디자인 스프린트에서 실제 인터랙션이 가능한 프로토타입이 필요할 때
- "이거 만들 수 있나?" 질문에 코드로 답하고 싶을 때

## 소요 시간

20-40분

## 사전 준비

- Claude Code 또는 AI 코딩 에이전트 설치
- 터미널 환경 (Node.js, Python 등 런타임)
- 아이디어 한 줄 설명 (구체적일수록 좋음)

## Step 1: 아이디어를 구조화된 스펙으로 변환하기

AI에게 바로 "만들어줘"라고 하면 원하는 결과가 나오기 어렵습니다. 핵심은 **30초 브리프**를 먼저 작성하는 것이에요.

| 항목 | 작성 예시 |
|------|----------|
| 한 줄 설명 | "GitHub 스타 트렌드를 추적하는 대시보드" |
| 핵심 기능 3개 | 레포 검색, 스타 그래프, 주간 리포트 |
| 기술 스택 | Next.js + Supabase |
| 타겟 사용자 | 오픈소스 메인테이너 |

```bash
# CLAUDE.md에 프로토타입 스펙을 먼저 정의
cat > CLAUDE.md << 'EOF'
# Project: github-star-tracker

## 목적
GitHub 레포의 스타 트렌드를 시각화하는 대시보드 MVP

## 핵심 기능
1. 레포 URL 입력 → 스타 히스토리 조회
2. 일별/주별 스타 증감 차트
3. 여러 레포 비교 뷰

## 기술 선택
- Next.js 15 (App Router)
- Recharts (차트)
- GitHub REST API (데이터)

## 제약 사항
- 인증 없이 동작 (Public API만 사용)
- 모바일 반응형 필수
- 초기 로딩 3초 이내
EOF
```

> **팁:** CLAUDE.md에 스펙을 적어두면 AI가 세션 내내 맥락을 유지합니다. 구두 설명보다 파일이 정확해요.

## Step 2: 스캐폴딩 — 뼈대를 한 번에 생성하기

AI에게 프로젝트 전체 구조를 한 번에 만들어 달라고 요청하세요. 파일 하나씩 만드는 것보다 훨씬 일관된 결과가 나옵니다.

```bash
# 프로젝트 초기화 + 의존성 설치까지 한 번에
claude "CLAUDE.md 스펙을 기반으로 프로젝트를 초기화해줘.
- package.json, tsconfig, 폴더 구조 생성
- 핵심 의존성 설치
- 홈 페이지 레이아웃 컴포넌트
- GitHub API 호출 유틸리티
- 빈 차트 컴포넌트 스켈레톤
한 번에 전부 만들어줘."
```

생성될 구조 예시:

```
github-star-tracker/
├── app/
│   ├── layout.tsx        # 루트 레이아웃
│   ├── page.tsx          # 메인 대시보드
│   └── api/
│       └── stars/route.ts  # API 프록시
├── components/
│   ├── RepoSearch.tsx    # 검색 입력
│   ├── StarChart.tsx     # 차트 컴포넌트
│   └── RepoCompare.tsx   # 비교 뷰
├── lib/
│   └── github.ts         # API 유틸리티
├── CLAUDE.md
└── package.json
```

> **핵심 원칙:** 스캐폴딩 단계에서는 완벽한 코드가 아니라 **올바른 구조**가 목표입니다. 세부 로직은 다음 단계에서 채워요.

## Step 3: 핵심 기능 하나를 끝까지 완성하기

3개 기능을 동시에 만들지 마세요. **가장 중요한 기능 하나**를 입력부터 화면 표시까지 완전히 동작하게 만드는 게 먼저입니다.

```bash
# 기능 1: 레포 검색 → 스타 히스토리 표시
claude "첫 번째 기능을 완성해줘:
1. RepoSearch에서 'owner/repo' 형식 입력 받기
2. GitHub API로 스타 히스토리 가져오기
3. StarChart에 일별 데이터 시각화
4. 로딩 상태와 에러 처리 포함
실제로 동작하는 코드를 작성해줘."
```

| 단계 | 확인 항목 |
|------|----------|
| 입력 | 레포 URL 입력 후 Enter → API 호출 |
| 처리 | 로딩 스피너 표시, 에러 시 메시지 |
| 출력 | 차트에 데이터 렌더링 |

```bash
# 실행해서 바로 확인
npm run dev
# 브라우저에서 localhost:3000 → 레포 입력 → 차트 확인
```

> **프로토타입의 핵심:** 하나가 동작하면 나머지는 빠릅니다. 첫 기능에서 데이터 흐름이 검증되니까요.

## Step 4: 나머지 기능을 빠르게 추가하기

첫 기능이 동작하면 AI가 프로젝트 컨텍스트를 이미 파악하고 있어서 나머지 기능 추가가 빨라집니다.

```bash
# 기능 2: 주간 증감 표시
claude "StarChart 아래에 주간 스타 증감을 테이블로 보여줘.
- 최근 4주 데이터
- 증감에 따라 초록/빨강 색상
- 기존 차트 컴포넌트와 스타일 일관성 유지"

# 기능 3: 레포 비교
claude "RepoCompare 컴포넌트를 완성해줘.
- 최대 3개 레포 추가/제거
- 하나의 차트에 여러 레포 데이터 오버레이
- 범례(legend)에 레포명 표시"
```

이 단계에서 유용한 프롬프트 패턴:

| 상황 | 프롬프트 예시 |
|------|-------------|
| UI 다듬기 | `"현재 페이지 스크린샷을 보고 UI를 개선해줘. 여백, 정렬, 반응형"` |
| 버그 수정 | `"npm run dev 에러 로그를 보고 수정해줘: [에러 붙여넣기]"` |
| 기능 확장 | `"기존 StarChart에 줌인/줌아웃 기능을 추가해줘"` |

## Step 5: 30초 스모크 테스트

프로토타입에 단위 테스트를 작성할 필요는 없지만, **핵심 경로가 동작하는지** 빠르게 확인하세요.

```bash
# AI에게 스모크 테스트 체크리스트 생성 요청
claude "이 프로젝트의 핵심 사용자 플로우를 체크리스트로 만들어줘.
실제로 브라우저에서 확인할 수 있는 항목만."
```

체크리스트 예시:

```markdown
## 스모크 테스트
- [ ] 페이지 로드 — 검색바 표시
- [ ] 'facebook/react' 입력 → 차트 렌더링
- [ ] 존재하지 않는 레포 입력 → 에러 메시지
- [ ] 모바일 화면 → 레이아웃 깨지지 않음
- [ ] 두 번째 레포 추가 → 비교 차트 표시
```

> **프로토타입 기준:** 5개 체크리스트 중 4개 이상 통과하면 데모 가능 수준입니다.

## Step 6: 정리 + 다음 단계 결정

프로토타입이 동작하면 두 가지 선택지가 있어요.

### 옵션 A: 버리고 다시 만들기 (추천)

프로토타입은 검증 도구입니다. 동작 확인 후 제대로 된 설계로 다시 시작하세요.

```bash
# AI에게 프로토타입에서 배운 점을 정리하게 하기
claude "이 프로토타입에서 발견한 것들을 정리해줘:
1. 기술적으로 어려웠던 부분
2. API 제약사항이나 예상 못한 문제
3. 프로덕션에서 달라져야 할 구조
LESSONS.md로 저장해줘."
```

### 옵션 B: 프로토타입을 발전시키기

빠르게 출시해야 하면 프로토타입을 정리해서 MVP로 전환합니다.

```bash
# 프로토타입 → MVP 전환
claude "이 프로토타입을 프로덕션 수준으로 정리해줘:
1. 에러 핸들링 강화
2. 환경 변수 분리
3. 기본 SEO 메타 태그
4. README.md 작성
새 코드를 추가하지 말고 기존 코드를 정리하는 데 집중해줘."
```

| 기준 | 버리기 | 발전시키기 |
|------|--------|----------|
| 검증 목적만 | ✅ | |
| 1주 내 출시 필요 | | ✅ |
| 구조 변경 필요 | ✅ | |
| 80% 이상 재사용 가능 | | ✅ |

## 체크리스트

- [ ] CLAUDE.md에 30초 브리프 작성 완료
- [ ] 프로젝트 스캐폴딩 생성 및 실행 확인
- [ ] 핵심 기능 1개 입력→출력 완전 동작
- [ ] 나머지 기능 추가 (최소 데모 가능 수준)
- [ ] 스모크 테스트 통과
- [ ] 다음 단계 결정 (버리기 vs 발전시키기)

## 프로토타이핑을 빠르게 하는 팁

| 팁 | 설명 |
|----|------|
| 스펙 먼저 | 코드 전에 CLAUDE.md를 30초만 투자해서 쓰세요 |
| 한 기능씩 | 3개 동시보다 1개 완성이 빠릅니다 |
| 서드파티 적극 사용 | 차트는 Recharts, 인증은 Supabase Auth — 직접 만들지 마세요 |
| 완벽함 버리기 | 프로토타입에서 코드 품질은 중요하지 않아요 |
| 즉시 실행 | 코드 작성 후 바로 `npm run dev`로 확인하는 습관 |
| 스크린샷 활용 | 현재 화면을 AI에게 보여주고 개선 요청하기 |
