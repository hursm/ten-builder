# PDF 구조화 & 요약 카드 관리 가이드

> 업무 서적 PDF를 Claude로 구조화하고, 요약 카드를 만들어 기획안에 활용하는 분들을 위한 실전 관리 가이드

## 전체 워크플로우

1. **원문 PDF 준비** - 큰 챕터별로 분할 (500p+ 용량 이슈 대응)
2. **마크다운 구조화** - Claude에게 "이 PDF 마크다운으로 바꿔줘"
3. **주제별 요약 카드 생성** - 챕터 순서가 아니라 주제/키워드 기준으로 카드화
4. **기획안 작성** - "이 주제 관련 카드 5개만 골라서 브리프 만들어줘"

> 2번에서 단순히 챕터 순서대로 나누는 것보다 **주제/키워드 태깅을 같이** 해두면 나중에 책이 쌓였을 때 크로스 검색이 됩니다.

## 추천 폴더 구조

```
book-cards/
├── 마케팅불변의법칙/
│   ├── 원본/           ← PDF 원본 보관
│   ├── 마크다운/        ← 챕터별 구조화 (.md)
│   └── 카드/            ← 요약 카드 (.md)
├── 린스타트업/
│   ├── 원본/
│   ├── 마크다운/
│   └── 카드/
├── ...
└── 마스터-인덱스.csv     ← 이게 핵심!
```

### 폴더 위치

- **macOS**: ~/Documents/book-cards/
- **Windows**: C:\Users\내이름\Documents\book-cards\

## 마스터 인덱스 (핵심)

폴더만 있으면 5권까지는 괜찮은데, 20권 넘어가면 검색이 안 됩니다. Google Sheets나 Numbers에 아래 형식으로 한 줄씩 기록하세요.

| 책 | 저자 | 챕터 | 카드 제목 | 태그 | 한줄 요약 | 파일 경로 | 추가일 |
|---|---|---|---|---|---|---|---|
| 마케팅 불변의 법칙 | 알 리스 | Ch3 | 카테고리 법칙 | #마케팅 #포지셔닝 | 새 카테고리 1등이 기존 시장 2등보다 낫다 | 카드/ch3-카테고리법칙.md | 2026-03-19 |
| 린 스타트업 | 에릭 리스 | Ch5 | MVP 정의 | #MVP #검증 | 최소 기능으로 가설 검증 | 카드/ch5-mvp정의.md | 2026-03-20 |

> 이 인덱스가 있으면 "포지셔닝 관련 카드 전부 찾아줘" 같은 요청이 가능해집니다.

## Claude Projects 활용법

Claude 앱(데스크탑/웹)의 **프로젝트** 기능을 활용하면 효율이 올라갑니다.

### Projects에 올려둘 것

- 카드 템플릿 (카드 형식 통일용)
- 태그 규칙 (어떤 태그를 쓸지 정의)
- 마스터 인덱스 최신본

### Projects에 올리지 않을 것

- 원본 PDF 전부 (용량/컨텍스트 한계)
- 모든 카드 파일 (필요할 때 대화에서 첨부)

> Projects는 "Claude에게 맥락을 주는 용도"이지 저장소가 아닙니다. 원본은 항상 로컬 폴더가 정본(source of truth)입니다.

### 카드 템플릿 예시

Projects에 아래를 올려두면 매번 형식을 설명할 필요 없습니다:

```markdown
## [카드 제목]

- **출처**: 책 제목 / 챕터
- **핵심 주장**: (1-2문장)
- **사례**: (있으면)
- **태그**: #태그1 #태그2
- **한줄 요약**: (기획안에 바로 쓸 수 있는 한 문장)
```

## GitHub는 왜 비추천인가

- PDF 파일 다루기 불편 (대용량 파일 제한)
- 버전 관리는 좋지만 진입 장벽이 높음
- 개발자가 아니면 터미널 작업이 부담
- 위의 폴더 + 인덱스 조합이 더 실용적

## 백업 전략 (중요!)

### 동기화와 백업의 차이

|  | 동기화 (iCloud/Google Drive) | 백업 (Time Machine) |
|---|---|---|
| 실시간 반영 | O | X (주기적) |
| 파일 삭제 시 | 같이 삭제됨 | 과거 버전 복구 가능 |
| 용도 | 여러 기기에서 접근 | 실수 복구 |

### 추천 조합

1. **정본**: 로컬 폴더 (~/Documents/book-cards/)
2. **동기화**: iCloud Drive 또는 Google Drive (여러 기기 접근용)
3. **백업**: Time Machine 자동 백업 (맥 기본 기능, 외장 하드 연결만 하면 됨)

> 3개 중 2번만 하면 "파일 실수로 지웠는데 구글 드라이브에서도 사라졌어요" 사고가 납니다. 반드시 독립 백업을 하나 더 두세요.

## 업그레이드 경로 (나중에 여유 생기면)

### Obsidian (무료, 마크다운 노트앱)

- 카드 간 링크 연결 (`[[카테고리 법칙]]` 식으로 참조)
- 태그 검색, 그래프 뷰
- 로컬 파일 기반이라 소유권 완전 보장
- 단점: 초기 셋업이 폴더보다 한 단계 복잡

### Notion (클라우드 기반)

- 데이터베이스 뷰가 강력 (표, 갤러리, 칸반)
- 비개발자에게 가장 친숙
- 단점: 데이터가 클라우드에 있어서 주기적 Export 권장

> 처음부터 도구에 투자하지 마세요. 폴더 + 인덱스로 3-5권 해보고, 불편한 점이 생기면 그때 Obsidian이나 Notion으로 옮겨도 늦지 않습니다. (어차피 마크다운 파일이라 이동이 쉽습니다)
