| **GPT-4o** | $2.50 | $10 | $1.25 | 128K | 범용 코딩 |

## 비용 절감 5대 전략

### 1️⃣ 모델 라우팅 — 작업별 최적 모델 선택

```
단순 작업 (자동완성, 포맷팅, 변수명)  → Haiku / Flash / 4.1 mini
일반 코딩 (함수 작성, 테스트, 리뷰)   → Sonnet / GPT-4o / 4.1
복잡한 작업 (아키텍처, 리팩토링)      → Opus / GPT-4.1 / Gemini Pro
```

**실전 설정 예시 (Claude Code `settings.json`):**
```json
{
  "model": "claude-sonnet-4-20250514",
  "smallModelOverride": "claude-haiku-3-5-20241022"
}
```

### 2️⃣ 프롬프트 캐싱 — 반복 컨텍스트 재활용

| 전략 | 설명 | 절감 효과 |
|------|------|----------|
| CLAUDE.md 고정 | 프로젝트 규칙을 앞에 배치 → 자동 캐싱 | 입력 90% ↓ |
| 파일 순서 고정 | 같은 파일을 같은 순서로 참조 | 캐시 히트 ↑ |
| 세션 유지 | `/compact` 대신 세션 연속 사용 | 재전송 방지 |
| 배치 실행 | 비슷한 작업 연속 실행 | 워밍업 1회 |

### 3️⃣ 컨텍스트 최적화 — 불필요한 토큰 줄이기

```bash
# .claudeignore 예시 — 불필요한 파일 제외
node_modules/
dist/
*.lock
*.min.js
coverage/
.next/
```

**컨텍스트 다이어트 체크리스트:**
- [ ] `node_modules`, `dist` 등 빌드 산출물 제외
- [ ] 대형 JSON/CSV 데이터 파일 제외
- [ ] 이미지, 바이너리 파일 제외
- [ ] 관련 없는 서브 디렉토리 제외
- [ ] 전체 파일 대신 관련 함수만 참조

### 4️⃣ 작업 단위 최적화 — 큰 작업 분할

| 패턴 | Bad (비용 높음) | Good (비용 낮음) |
|------|---------------|-----------------|
| 범위 | "전체 앱 리팩토링해줘" | "auth 모듈만 리팩토링해줘" |
| 지시 | "알아서 해줘" | "이 함수의 반환 타입을 Promise<User>로 변경해줘" |
| 반복 | 매번 전체 컨텍스트 재전송 | `/compact` 후 핵심만 유지 |
| 검증 | AI에게 반복 수정 요청 | 로컬 테스트 후 한 번에 피드백 |

### 5️⃣ 구독 vs API — 상황별 최적 선택

| 구독 플랜 | 월 비용 | 포함량 | 적합한 경우 |
|----------|--------|-------|-----------|
| Claude Pro | $20 | Sonnet 무제한급 | 개인 사이드 프로젝트 |
| Claude Max (5x) | $100 | Opus 사용 가능 | 주력 개발 도구 |
| Claude Max (20x) | $200 | Opus 대량 사용 | 풀타임 AI 코딩 |
| Claude Code API | 종량제 | 사용한 만큼 | 팀/CI/자동화 |
| Cursor Pro | $20 | 500 요청 | IDE 통합 선호 |
| GitHub Copilot | $10-$19 | 무제한 자동완성 | 자동완성 중심 |

## 비용 절감 공식

```
월 비용 = Σ (토큰 수 × 단가) - 캐시 절감
       = (입력 토큰 × 입력 단가 × (1 - 캐시율))
       + (출력 토큰 × 출력 단가)

예시) 하루 Sonnet 50만 입력 + 10만 출력, 캐시율 60%
     = 500K × $3/M × 0.4 + 100K × $15/M
     = $0.60 + $1.50 = $2.10/일 ≈ $63/월
```

## 안티패턴 🚫

| 안티패턴 | 비용 영향 | 대안 |
|---------|----------|------|
| 모든 작업에 Opus 사용 | 5x~10x 초과 | 단순 작업은 Haiku로 |
| 전체 레포를 컨텍스트에 넣기 | 입력 토큰 폭발 | 관련 파일만 선택적 포함 |
| 같은 실수를 반복 피드백 | 출력 토큰 낭비 | 명확한 스펙 + 테스트 먼저 |
| 세션을 자주 새로 시작 | 캐시 무효화 | 세션 유지 + `/compact` |
| CI에서 매 커밋마다 AI 리뷰 | API 비용 급증 | PR 단위로만 실행 |
