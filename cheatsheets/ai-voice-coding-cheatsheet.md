# AI 보이스 코딩 치트시트 — 말로 개발하기

> 말이 타이핑보다 3배 빠르다 — AI 코딩 도구에 음성 입력을 연결하면 생각의 속도로 코딩할 수 있다.

## Claude Code 음성 모드 빠른 시작

| 동작 | 방법 |
|------|------|
| 음성 모드 활성화 | `/voice` 입력 |
| 말하기 시작 | 스페이스바 누른 상태 유지 |
| 말하기 종료 | 스페이스바 떼기 |
| 음성 모드 종료 | `/voice` 재입력 또는 `Ctrl+C` |
| Plan Mode + 음성 | `/plan` 실행 후 스페이스바로 구조 설명 |

> Claude Code 음성 모드는 2026년 3월 3일 배포됨. `claude --version` 최신 버전 확인 필수.

## Willow Voice 연동 (macOS)

```bash
# Willow 설치 후 Claude Code 세션에서 자동 활성화
# 글로벌 단축키로 어디서든 딕테이션 시작
brew install --cask willow

# 설정: 시스템 환경설정 → Willow → Claude Code 통합 활성화
```

| 설정 항목 | 권장값 |
|----------|--------|
| 마이크 | 노이즈 캔슬링 헤드셋 |
| 언어 | 영어 (기술 용어 인식률 높음) |
| 단축키 | `Option+Space` |
| 자동 전송 | 2초 침묵 후 전송 |

## 음성 프롬프트 패턴

### 기능 구현 요청

```
# 일반 타이핑
"Add a function that validates email addresses using regex"

# 음성으로 말하기 (더 자연스럽게)
"클래스에 이메일 유효성 검사 메서드 추가해줘.
 정규식 쓰고, 잘못된 형식이면 에러 메시지 반환해"
```

### 리팩토링 요청

```
"이 함수 너무 길어 — 50줄 이하로 쪼개고
 단일 책임 원칙 적용해서 읽기 쉽게 만들어줘"
```

### 에러 해결

```
"지금 터미널에 타입 에러 나오는데
 어떤 타입을 기대하는지 설명하고 수정해줘"
```

### 코드 리뷰 요청

```
"방금 작성한 코드 리뷰해줘.
 보안 문제, 엣지 케이스, 성능 이슈 위주로"
```

## 생산성 높이는 음성 코딩 팁

| 상황 | 팁 |
|------|-----|
| 복잡한 아키텍처 설명 | Plan Mode에서 전체 흐름을 먼저 말로 설명 |
| 코드 네이밍 | 영어로 말하는 게 변수명·함수명 인식 정확도 높음 |
| 반복 작업 | 첫 번째만 음성으로 설명 → 패턴 적용 요청 |
| 문서 작성 | JSDoc, docstring 음성으로 빠르게 구술 |
| 커밋 메시지 | 변경 내용 말로 설명 → AI가 conventional commit 형식으로 |

## 흔한 실수와 해결

| 실수 | 해결 |
|------|------|
| 기술 용어 오인식 | 영어 발음 명확히 + 철자 말하기 ("useState, u-s-e-S-t-a-t-e") |
| 너무 긴 문장 | 한 요청을 2~3 문장으로 나눠서 말하기 |
| 배경 소음 간섭 | 노이즈 캔슬링 마이크 + 조용한 환경 |
| 코드 실행 중 음성 입력 | 실행 완료 후 다음 요청 |
| 컨텍스트 없이 수정 요청 | "지금 열려있는 파일에서..." 로 시작 |

## 음성 코딩에 적합한 작업 vs 비적합한 작업

| 적합 | 비적합 |
|------|--------|
| 기능 요청, 아이디어 설명 | 정밀 인라인 편집 |
| 리팩토링 방향 지시 | 복잡한 정규식 구술 |
| 문서 작성, 커밋 메시지 | 코드 파일 직접 딕테이션 |
| Plan Mode 구조 설계 | 특수문자 많은 코드 |
| 에러 증상 설명 | 짧은 한 단어 명령 |

## 음성 + 키보드 하이브리드 워크플로우

```
1. [음성] 기능 요청 → Claude Code 초안 생성
2. [키보드] 생성된 코드 빠르게 검토
3. [음성] "이 부분 수정해줘, {설명}"
4. [키보드] 최종 확인 + 커밋
5. [음성] "커밋 메시지 써줘"
```
