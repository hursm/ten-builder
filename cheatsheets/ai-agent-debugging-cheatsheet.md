# AI 에이전트 디버깅 치트시트

> AI 코딩 에이전트가 기대와 다르게 동작할 때 — 원인 파악부터 해결까지 한 페이지 요약

## 에이전트 실패 유형 5가지

| 유형 | 증상 | 핵심 원인 |
|------|------|----------|
| 컨텍스트 유실 | 이전 대화 내용을 무시하고 엉뚱한 코드 생성 | 토큰 한도 초과, 컨텍스트 윈도우 넘침 |
| 도구 오용 | 파일을 잘못된 경로에 쓰거나, 없는 명령어 실행 | 도구 스키마 불일치, 모호한 지시 |
| 무한 루프 | 같은 수정을 반복하거나, 빌드-에러-수정 사이클 탈출 불가 | 에러 패턴 인식 실패, 탈출 조건 미설정 |
| 환각(Hallucination) | 존재하지 않는 API, 라이브러리, 함수 사용 | 학습 데이터 시점 차이, 그라운딩 부족 |
| 부분 완료 | 요청의 일부만 수행하고 "완료"라고 보고 | 복잡한 요청 분해 실패, 체크리스트 부재 |

## 증상별 즉시 대응

### 컨텍스트 유실 대응

```bash
# Claude Code: 컨텍스트 확인
/compact  # 대화 요약 후 윈도우 확보

# 핵심 파일만 명시적으로 지정
"@src/api/routes.ts 와 @src/models/user.ts 만 참고해서 수정해줘"
```

**체크포인트:**
- 대화가 50턴 이상이면 → `/clear` 후 핵심 요구사항 재정리
- CLAUDE.md에 프로젝트 규칙을 넣어두면 매번 반복 설명 불필요

### 도구 오용 대응

```bash
# 에이전트가 실행할 명령어를 먼저 확인
"실행 전에 계획을 보여줘. 승인하면 그때 실행해."

# 도구 권한 제한 (Claude Code)
claude --allowedTools "Edit,Read,Bash(git *)"
```

| 상황 | 프롬프트 |
|------|---------|
| 잘못된 파일 수정 시 | `"방금 수정을 되돌려줘. {파일명}만 변경해야 해"` |
| 위험한 명령어 실행 시 | `"rm, drop, delete 명령어는 사용하지 마"` |
| 경로 혼동 시 | `"프로젝트 루트는 /app이야. 절대경로 기준으로 작업해"` |

### 무한 루프 탈출

```bash
# 3회 연속 같은 에러 → 에이전트 중단 후 수동 개입
# Ctrl+C로 중단

# 재시작 시 컨텍스트 리셋
"이전 시도는 잊어줘. 에러 메시지는 이거야: {에러}
다른 접근 방식으로 해결해봐."
```

**무한 루프 방지 패턴:**
- 최대 재시도 횟수를 프롬프트에 명시: `"3번까지만 시도해"`
- 실패 시 대안 요청: `"이 방식이 안 되면 다른 라이브러리를 제안해줘"`

### 환각 검증

```bash
# 생성된 패키지가 실제 존재하는지 확인
npm info {패키지명} 2>/dev/null || echo "존재하지 않는 패키지"
pip show {패키지명} 2>/dev/null || echo "설치되지 않음"

# API 엔드포인트 검증
curl -s -o /dev/null -w "%{http_code}" https://api.example.com/v2/endpoint
```

| 환각 유형 | 검증 방법 |
|----------|----------|
| 없는 패키지 | `npm info` / `pip show`로 확인 |
| 잘못된 API | 공식 문서 URL 직접 확인 |
| 없는 함수 | IDE 자동완성으로 교차 확인 |
| 틀린 옵션 | `--help` 플래그로 실제 옵션 확인 |

### 부분 완료 방지

```markdown
# 요청 시 체크리스트 형태로 구조화
다음 항목을 모두 완료해줘:
- [ ] API 엔드포인트 추가 (/api/users)
- [ ] 입력 검증 미들웨어 작성
- [ ] 에러 핸들링 추가
- [ ] 단위 테스트 작성
- [ ] 타입 정의 업데이트

각 항목 완료할 때마다 체크 표시하고 진행 상황 알려줘.
```

## 에이전트별 디버깅 도구

| 에이전트 | 로그 확인 | 컨텍스트 리셋 | 도구 제한 |
|---------|----------|-------------|----------|
| Claude Code | `~/.claude/logs/` | `/clear` | `--allowedTools` |
| Cursor | 출력 패널 확인 | 새 대화 시작 | Rules 파일 |
| Copilot | 출력 채널 로그 | `#clear` | `.github/copilot-instructions.md` |
| Aider | `--verbose` 플래그 | `/clear` | `.aider.conf.yml` |
| Cline | 출력 패널 | 새 대화 | 설정에서 제한 |

## 구조적 디버깅 워크플로우

```
1. 증상 식별
   └─ 위 5가지 유형 중 어디에 해당?

2. 로그 수집
   └─ 에이전트 로그 + 에러 메시지 + 실행 이력

3. 컨텍스트 확인
   └─ 토큰 사용량이 한계 근처인가?
   └─ 핵심 파일이 컨텍스트에 포함되어 있는가?

4. 격리 테스트
   └─ 새 세션에서 동일 요청 재현 시도
   └─ 프롬프트를 단순화해서 테스트

5. 수정 적용
   └─ 프롬프트 개선 / 도구 제한 / 컨텍스트 리셋
```

## 예방 패턴: CLAUDE.md 디버깅 규칙 예시

```markdown
# 디버깅 규칙
## 금지
- rm -rf, DROP TABLE 등 파괴적 명령어 실행 금지
- 3회 동일 에러 반복 시 중단하고 보고

## 필수
- 파일 수정 전 현재 상태를 git stash로 백업
- 외부 패키지 추가 시 버전 명시
- 빌드 에러 발생 시 전체 로그 출력 후 분석
```

## 흔한 실수 & 해결

| 실수 | 왜 발생하는가 | 해결 |
|------|-------------|------|
| "모든 파일을 한 번에 수정해줘" | 변경 범위가 넓으면 실수 확률 증가 | 파일 단위로 나눠서 요청 |
| 에이전트에 맥락 없이 에러만 전달 | 에이전트가 코드베이스 구조 모름 | 관련 파일 경로 + 에러 로그 함께 전달 |
| 실패 후 같은 프롬프트 재시도 | 같은 입력이면 같은 출력 | 접근 방식 변경을 명시적으로 요청 |
| 에이전트 출력을 검증 없이 적용 | 환각 가능성 존재 | `git diff` 리뷰 + 테스트 실행 후 커밋 |
| 긴 세션에서 계속 작업 | 컨텍스트 오염 누적 | 주기적으로 새 세션 시작 |
