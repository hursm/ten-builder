# PIV 루프 에이전틱 코딩 치트시트

> Plan → Implement → Validate — 바이브 코딩 대신 구조화된 AI 개발 워크플로우

---

## PIV 루프란?

AI 코딩 에이전트에게 충분한 자율성을 주면서도 개발자가 제어권을 유지하는 3단계 반복 워크플로우입니다.

```
Plan ──→ Implement ──→ Validate
  ↑____________________________|
```

---

## Phase 1: Plan (계획)

**목표:** 코드 작성 전 AI와 함께 명확한 스펙 정의

| 항목 | 예시 |
|------|------|
| 태스크 범위 | "인증 미들웨어 추가 (JWT, 만료 처리)" |
| 제약 조건 | "기존 Express 라우터 구조 유지" |
| 완료 기준 | "단위 테스트 통과, 기존 엔드포인트 미영향" |
| 금지 사항 | "외부 라이브러리 추가 금지" |

### Plan 단계 프롬프트 패턴

```bash
# 계획 수립 프롬프트
claude "다음 태스크를 구현하기 전 계획을 세워줘:
- 목표: [태스크 설명]
- 제약: [조건들]
- 완료 기준: [검증 방법]
코드 작성 전 계획만 출력하고 내 확인을 기다려줘."
```

### Plan 체크리스트

- [ ] 태스크가 하나의 PR에 담길 만큼 작은가?
- [ ] 완료 기준이 검증 가능한가?
- [ ] 영향 범위(파일/모듈)가 명확한가?
- [ ] 롤백 방법이 있는가?

---

## Phase 2: Implement (구현)

**목표:** 계획된 스펙 내에서 AI가 코드 생성

| 자율성 수준 | 사용 시점 | 예시 |
|------------|----------|------|
| 높음 (Auto) | 단순 CRUD, 보일러플레이트 | `"계획대로 구현해줘"` |
| 중간 (Guided) | 비즈니스 로직 포함 | 단계별 확인하며 진행 |
| 낮음 (Manual) | 보안/결제 관련 | 한 파일씩 검토 후 진행 |

### 구현 단계 핵심 패턴

```bash
# 체크포인트 방식 (장기 구현)
claude "계획 기반으로 구현해줘.
각 파일 작업 완료 후 잠깐 멈추고 요약해줘.
다음 파일로 가기 전 내가 확인할게."

# 한 번에 구현 (단순 태스크)
claude "계획대로 구현해줘. 테스트 포함."
```

### 구현 중 주의 사항

| 신호 | 대응 |
|------|------|
| 에이전트가 스펙 범위를 벗어남 | 즉시 중단, 범위 재확인 |
| 예상보다 파일 수정이 많음 | Plan으로 돌아가기 |
| "이 부분도 고쳐야 할 것 같아요" | 새 태스크로 분리 |
| 컨텍스트 창 80% 초과 | 새 세션으로 재시작 |

## Phase 3: Validate (검증)

**목표:** Plan 단계 완료 기준 대비 구현 결과 검증

### 자동화 검증

```bash
# 테스트 실행
npm test -- --coverage

# 타입 체크
npx tsc --noEmit

# 린트
npx eslint src/ --ext .ts,.tsx

# 변경 파일 확인
git diff --stat
```

### AI 보조 검증 프롬프트

```bash
# 코드 리뷰 요청
claude "구현한 코드를 Plan 단계 완료 기준과 비교해서
통과/미통과를 체크리스트로 알려줘."

# 엣지 케이스 확인
claude "이 구현에서 실패할 수 있는 엣지 케이스 5가지를 찾아줘."
```

### Validate 체크리스트

- [ ] 단위 테스트 통과
- [ ] 기존 테스트 미깨짐
- [ ] 완료 기준 항목 충족
- [ ] 보안 이슈 없음 (시크릿 노출, SQL 인젝션 등)
- [ ] diff 크기가 예상 범위 내 (±20%)

---

## PIV 루프 반복 전략

```
작은 태스크 (< 1시간)
  → 루프 1회로 완료

중간 태스크 (< 1일)
  → 루프 2~3회, 각 루프마다 PR

큰 기능 (> 1일)
  → Stacked PR + 루프별 독립 브랜치
```

---

## 바이브 코딩 vs PIV 루프

| | 바이브 코딩 | PIV 루프 |
|--|------------|---------|
| 속도 | 빠름 (초반) | 보통 |
| 결과 예측성 | 낮음 | 높음 |
| 프로덕션 안전성 | 위험 | 안전 |
| 디버깅 시간 | 길어짐 | 짧음 |
| 팀 협업 | 어려움 | 쉬움 |
